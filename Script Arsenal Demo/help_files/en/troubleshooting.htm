1. In the Windows environment, dragging a script dialog across the document window may cause artifacts shown on screenshot 2. To get rid of those, just click the "Refresh" button.

2. In some cases, launching the mentioned scripts from the "Script Arsenal" dialog may cause Photoshop losing focus (in other words, some other application's window becoming the front one). To avoid this, run the problem scripts from either the menu or the Actions panel.

3. Some users report that any script that opens a dialog crashes with "Unable to create Window" error message:

Unfortunately, this is a bug in Adobe's script engine known to users of Photoshop CS3 and CS4. As a rule, this problem resolves itself after several computer re-boots. However, if in your case the problem persists, the author can suggest the only solution - using the scripts that don't open dialogs (to get to know which ones, please, see the documentation). To launch such a script, either add it to "File > Scripts" menu (by copying the proper JSX files to <Photoshop>/Presets/Scripts" folder), or by creating an action.
